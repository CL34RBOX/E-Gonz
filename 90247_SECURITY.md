# Security Policy

## Supported Versions
We provide security updates for actively maintained branches. Older branches may not receive patches.

| Version | Supported |
|---------|-----------|
| main    | Yes       |
| dev     | Yes       |
| legacy  | No        |

## Reporting a Vulnerability
If you discover a vulnerability, please report it responsibly.

- Email: security@example.com
- Include: steps to reproduce, impact, affected components, logs if available
- We will acknowledge receipt within **48 hours**

## Disclosure Policy
We follow coordinated disclosure:

1. Researcher privately reports the issue  
2. We validate and assign a severity  
3. We develop and test a fix  
4. Researcher is notified before public disclosure  
5. Advisory is published after patch release  

## Scope
In-scope:
- Code in this repository
- Configuration files
- Build and deployment scripts

Out-of-scope:
- Third-party dependencies
- User environment issues
- Social engineering

## Response Timeline
- **48 hours**: Acknowledgement  
- **5 business days**: Initial assessment  
- **10 business days**: Fix or mitigation plan  

Thank you for helping keep this project secure.
